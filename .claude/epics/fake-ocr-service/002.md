# Task: Core constants

## Description

Create `app/core/config.py` containing all module-level constants for the service: engine version, upload size limit, supported extensions set, and the per-extension response template strings. This is the single source of truth for the values that the schemas, the service, and the router will read — no values are hardcoded in those downstream modules. Implements the constant side of [[2026-04-26-fake-ocr-engine-constraint]].

## Acceptance Criteria

- [ ] `app/core/config.py` defines:
  - `ENGINE_VERSION: str = "fake-v0"`
  - `MAX_UPLOAD_BYTES: int = 10 * 1024 * 1024`  (10 MiB; PRD §FR-6)
  - `SUPPORTED_EXTENSIONS: frozenset[str] = frozenset({"pdf", "png", "txt"})`
  - `RESPONSE_TEMPLATES: dict[str, str]` with keys `"pdf"`, `"png"`, `"txt"` and the exact PRD §FR-2 template strings, each containing a `{filename}` placeholder.
- [ ] Templates match PRD §FR-2 byte-for-byte:
  - `"pdf"` → `"Mock OCR result for PDF document.\nFilename: {filename}\nPages: 3"`
  - `"png"` → `"Mock OCR result for PNG image.\nFilename: {filename}\nDimensions: 1024x768"`
  - `"txt"` → `"Mock OCR result for text file.\nFilename: {filename}\nLines: 42"`
- [ ] No env-var lookups, no `Settings` class, no `os.getenv`. Module-level constants only (PRD §Constraints).
- [ ] **Gate (mirrors Phase B):** `uv run python -c "from app.core import config; print(config.ENGINE_VERSION)"` prints `fake-v0`.

## Technical Details

### Files to create

- `app/core/config.py`

### Notes

- `SUPPORTED_EXTENSIONS` is a `frozenset` to make it explicit that this is a contract, not a runtime-mutable setting.
- The `Pages: 3` and `Dimensions: 1024x768` in templates are template literals, not measured values. PRD §Out of Scope.
