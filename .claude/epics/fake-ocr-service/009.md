# Task: Integration tests

## Description

Build the integration test suite that exercises the assembled app via FastAPI's `TestClient` (sync, in-process). Covers happy paths, all four error codes, the determinism contract, and request-ID echo behaviour. Earns the ≥95% coverage gate and the determinism guarantee from [[2026-04-26-fake-ocr-engine-constraint]] + envelope conformance from [[2026-04-26-api-response-envelope-shape]].

## Acceptance Criteria

### `tests/integration/test_health_endpoint.py`

- [ ] `GET /v1/health` returns status 200.
- [ ] Response body equals `{"success": true, "data": {"status": "ok", "engine": "fake-v0"}, "error": null}`.
- [ ] Response includes `X-Request-ID` header (any value).
- [ ] Two-call test confirms a generated request-ID differs across requests when client sends none.

### `tests/integration/test_ocr_endpoint.py`

- [ ] **Happy path** test per supported extension (`pdf`, `png`, `txt`): exact body match against the PRD §FR-2 template (with the test's basename interpolated).
- [ ] **`no_file` (400)**: POST with no `file` part → status 400, `error.code == "no_file"`, message exactly `"No file part 'file' in request"`.
- [ ] **`missing_extension` (415)**: POST a file named `README` → status 415, `error.code == "missing_extension"`, message `"File 'README' has no extension; cannot determine handler."`.
- [ ] **`unsupported_extension` (415)**: POST a file named `data.xyz` → status 415, `error.code == "unsupported_extension"`, message `"Extension '.xyz' is not supported. Allowed: .pdf, .png, .txt"`.
- [ ] **`file_too_large` (413)**: POST a file ≥ 10 MB + 1 byte → status 413, `error.code == "file_too_large"`, message matches the `"File size <X> MB exceeds 10 MB limit"` shape.
- [ ] **Compound extension**: POST `archive.tar.gz` → 415 `unsupported_extension` (proves last-segment-only matching per PRD §FR-4).
- [ ] **Case-insensitive extension**: POST `INVOICE.PDF` → 200 success with `extension == "pdf"` in the response data.
- [ ] **Path-stripping**: POST a file whose multipart `filename` includes path components (e.g. `subdir/invoice.pdf`) → response `data.filename == "invoice.pdf"` with no path component.

### `tests/integration/test_determinism.py`

- [ ] For each supported extension, perform 50 sequential `POST /v1/ocr` calls with the same `(filename, bytes)` input and assert all 50 response bodies are byte-identical.
- [ ] The 50-call determinism check executes in well under 30 seconds (PRD success criterion 5 — the whole suite must complete in under 30 s on a fresh clone).

### `tests/integration/test_request_id.py` (or merged into health/ocr files)

- [ ] If client sends `X-Request-ID: <value>`, server echoes the exact same `<value>` in the response header. Tested on both `/v1/health` and `/v1/ocr`.
- [ ] If client omits `X-Request-ID`, server returns a generated UUIDv4-hex value and the header is present.

### Belt-and-braces test (per epic Success Criterion 10)

- [ ] One test that imports `app` and asserts `pytesseract`, `paddleocr`, `easyocr` are NOT in `sys.modules`. Locks [[2026-04-26-fake-ocr-engine-constraint]] mechanically.

### Gate (mirrors Phase J)

- [ ] `uv run pytest -v` exits zero with all tests above present.
- [ ] `pytest --cov=app --cov-fail-under=95` passes (95% threshold enforced via `pyproject.toml`).
- [ ] Total suite runtime < 30 seconds on the developer's machine.

## Technical Details

### Files to create

- `tests/integration/test_health_endpoint.py`
- `tests/integration/test_ocr_endpoint.py`
- `tests/integration/test_determinism.py`
- `tests/integration/test_request_id.py` (or fold its tests into the two endpoint files; pick one)
- `tests/integration/test_no_real_ocr_imports.py` (the belt-and-braces test; small and standalone)
- A `tests/conftest.py` if shared fixtures are needed (e.g., a `client` fixture wrapping `TestClient(app)`).

### Notes

- Use FastAPI's sync `TestClient` (which wraps `httpx.Client`). PRD §Constraints commits to this; subprocess-uvicorn E2E is a future ADR.
- Construct the 10 MB+1 byte payload in-memory (`b"x" * (10 * 1024 * 1024 + 1)`); do not write to disk.
- For determinism, hash each response body and assert all 50 hashes are equal — clearer failure messages than 50 string equalities.
- The 30-second budget is real. If the determinism test alone takes >5 s, profile before adding more cases.
