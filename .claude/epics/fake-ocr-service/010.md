# Task: CI workflow

## Description

Add a GitHub Actions workflow that runs `make check` on every push and pull request, on Ubuntu, on Python 3.11. This wires the entire epic's gate into CI. Wired up only after the local suite is green (depends on 009) so the first CI run reflects a known-good state — diagnosing CI failures and code failures simultaneously is a needless multiplication of variables.

## Acceptance Criteria

- [ ] `.github/workflows/ci.yml` exists with:
  - Triggers: `push` and `pull_request` (no branch filter — runs on every branch).
  - Single job, `runs-on: ubuntu-latest`.
  - Python 3.11 set up.
  - `uv` installed via `astral-sh/setup-uv@v3` (or current pinned major version).
  - `uv sync --frozen` step (uses the committed `uv.lock`; fails if drift exists).
  - `make check` step.
- [ ] No matrix (single Python version per PRD §Constraints).
- [ ] No deployment, no release, no artifact upload — CI is verification only in v1.
- [ ] **Gate (mirrors Phase K):** workflow runs green on the first push of the epic branch.

## Technical Details

### Files to create

- `.github/workflows/ci.yml`

### Notes

- `uv sync --frozen` rather than `uv sync` is intentional: CI must fail loudly if `pyproject.toml` and `uv.lock` disagree, rather than silently regenerating.
- Use `astral-sh/setup-uv@v3` and let it install Python from `.python-version` rather than calling `actions/setup-python` separately. One source of truth for the Python version.
- Cache `uv`'s download cache via the `enable-cache: true` input on the setup-uv action.
- Do not skip CI on doc-only changes for v1. The matrix is too small to justify path filtering.
