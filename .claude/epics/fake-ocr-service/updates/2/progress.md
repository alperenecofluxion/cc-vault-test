## Decisions made autonomously

1. **Dev deps via `[dependency-groups] dev`** (PEP 735) rather than `[project.optional-dependencies]`. Both satisfy the spec's "dev/optional group" wording; PEP 735 is the modern uv-preferred pattern and is installed by `uv sync` by default. Switching later costs one config rewrite.
2. **`[tool.uv] package = false`** — this is an application, not a library. Avoids the need for a `[build-system]` and avoids `uv` trying to build a wheel of the project itself. Cleanest setup for a service.
3. **Ruff config:** `line-length = 100`, `target-version = "py311"`, lint rules `["E", "F", "I", "B", "UP", "N", "W"]`. Production-shaped middle ground; not so strict that it becomes a rules debate, not so loose that lint gives up. The PRD spec required ruff to be "configured" but did not pin specific values.
4. **Dependency lower bounds:** Used recent-but-relaxed (`fastapi>=0.115`, `pytest>=8.3`, etc.). uv resolved to current latest. Lockfile freezes the exact version set; lower bounds only matter if someone regenerates the lockfile.
5. **Did not create a git worktree.** Per session instruction during `/pm:epic-sync`, working on `main` directly.
