# Task: OCR service and unit tests

## Description

Implement the deterministic mock OCR function in `app/services/ocr.py` and its unit tests in `tests/unit/test_ocr_service.py`. This is the literal implementation of [[2026-04-26-fake-ocr-engine-constraint]] — a pure function, no model, no I/O, no randomness. The unit tests prove determinism at the function level before the router is wired up.

## Acceptance Criteria

### Service (`app/services/ocr.py`)

- [ ] Exposes `run_ocr(basename: str, extension: str) -> str`.
- [ ] Implementation: `return RESPONSE_TEMPLATES[extension.lower()].format(filename=basename)`. That is the entire body.
- [ ] No filesystem access, no network access, no logging side effects, no `time.now()`, no `uuid`, no randomness.
- [ ] No exception handling for unsupported extensions — the caller (router) is responsible. This is documented in the module docstring.

### Unit tests (`tests/unit/test_ocr_service.py`)

- [ ] One test per supported extension (`pdf`, `png`, `txt`) that asserts the exact returned string against the PRD §FR-2 template, with a representative basename.
- [ ] One determinism test: 50 sequential calls with the same `(basename, extension)` produce 50 identical strings.
- [ ] One filename-interpolation test: confirms the basename is interpolated literally into the template (no escaping, no truncation, no path-component stripping at this layer — that's the router's responsibility).
- [ ] One case-insensitivity test: `run_ocr("a.pdf", "PDF")` produces the same string as `run_ocr("a.pdf", "pdf")`.

### Gate (mirrors Phases E + F-1)

- [ ] `uv run pytest tests/unit -v` exits zero with all tests above present and passing.

## Technical Details

### Files to create

- `app/services/ocr.py`
- `tests/unit/test_ocr_service.py`

### Notes

- Keep the service file tiny — ideally a docstring plus the one-line return. Resist the urge to add validation, logging, or exception types. The constraint is in [[2026-04-26-fake-ocr-engine-constraint]].
- The case-insensitivity test belongs at the service layer because `extension.lower()` is implemented inside `run_ocr`. The router will also lowercase before lookup; that's defence in depth, not duplication.
