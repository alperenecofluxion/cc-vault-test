# Task: OCR router

## Description

Implement `POST /v1/ocr` in `app/routers/ocr.py`. Handles multipart upload, basename + extension extraction, size and extension validation, and dispatch into the OCR service. Emits the canonical four error codes documented in the epic and PRD ┬¦US-2. Applies [[2026-04-26-fake-ocr-engine-constraint]] (calls the pure function), [[2026-04-26-api-response-envelope-shape]] (returns envelope on every code path), and [[2026-04-26-layered-project-structure]] (routing logic lives here, not in `main.py`).

## Acceptance Criteria

- [ ] `app/routers/ocr.py` exposes `router = APIRouter(prefix="/v1")`.
- [ ] Route: `POST /ocr` accepts `file: UploadFile = File(...)` (multipart/form-data, single part named `file`).
- [ ] Validation sequence (in this order ŌĆö order matters because earlier checks mask later failure modes):
  1. **No file part** ŌåÆ `400`, `error.code = "no_file"`, message `"No file part 'file' in request"`.
  2. **Read body, measure size; size > `MAX_UPLOAD_BYTES`** ŌåÆ `413`, `error.code = "file_too_large"`, message `"File size <observed_mb> MB exceeds 10 MB limit"` where `<observed_mb>` is the observed size in MB to one decimal place.
  3. **Strip path components** from `file.filename` (use `os.path.basename` or `pathlib.PurePath`) ŌĆö never echo a raw path back to the client (PRD ┬¦FR-5).
  4. **No extension** (no `.` in basename, or basename ends with `.`) ŌåÆ `415`, `error.code = "missing_extension"`, message `"File '<basename>' has no extension; cannot determine handler."`.
  5. **Lowercase the extension; not in `SUPPORTED_EXTENSIONS`** ŌåÆ `415`, `error.code = "unsupported_extension"`, message `"Extension '.<ext>' is not supported. Allowed: .pdf, .png, .txt"`.
  6. **Happy path** ŌåÆ call `run_ocr(basename, extension)`, return `SuccessResponse[OCRData](data=OCRData(text=..., extension=extension, filename=basename))` with status `200`.
- [ ] Errors return via `JSONResponse(status_code=..., content=ErrorResponse(...).model_dump())` so HTTP status and envelope agree.
- [ ] `Content-Type` header on the upload part is **not** validated (PRD ┬¦FR-6).
- [ ] Compound extensions (e.g. `archive.tar.gz`) match the LAST extension only (`gz` ŌåÆ unsupported).
- [ ] Extension matching is case-insensitive (`.PDF` ŌēĪ `.pdf`).
- [ ] **Gate (mirrors Phase H, partial):** module imports without error; routes can be mounted on a bare `FastAPI()` and the OpenAPI schema generates.

### Notes

- The size check requires reading the full body (PRD ┬¦FR-6: "Read-and-discard the full body"). Use `await file.read()`.
- The `<observed_mb>` formatting must be stable: round to one decimal place using `f"{observed / (1024*1024):.1f}"`. This is testable in 009.
- Defence-in-depth on extension lowercasing: the router lowercases before checking `SUPPORTED_EXTENSIONS`, even though `run_ocr` also lowercases internally. PRD ┬¦FR-4 puts the normalisation rule at the boundary.
- Return success via the typed `response_model=SuccessResponse[OCRData]` so the OpenAPI spec is correct; errors bypass `response_model` via raw `JSONResponse`.
