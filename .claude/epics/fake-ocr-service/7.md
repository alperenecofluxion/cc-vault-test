# Task: Health router

## Description

Implement `GET /v1/health` in `app/routers/health.py` returning a constant success envelope. No I/O, no liveness probes, no dependency checks — PRD §US-3 explicitly limits scope to "is the process alive enough to answer". Implements PRD §US-3 and applies [[2026-04-26-api-response-envelope-shape]] + [[2026-04-26-layered-project-structure]].

## Acceptance Criteria

- [ ] `app/routers/health.py` exposes a module-level `router = APIRouter(prefix="/v1")`.
- [ ] Route: `GET /health` → returns `SuccessResponse[HealthData](data=HealthData(status="ok"))`.
- [ ] Route declaration uses `response_model=SuccessResponse[HealthData]` so FastAPI enforces the envelope in the OpenAPI schema.
- [ ] Endpoint performs zero I/O, zero external calls, zero conditional logic. It is a pure constant return.
- [ ] **Gate (mirrors Phase H, partial):** the router can be mounted on a bare `FastAPI()` instance and `TestClient.get("/v1/health")` returns:
  - status code `200`
  - body `{"success": true, "data": {"status": "ok", "engine": "fake-v0"}, "error": null}`

## Technical Details

### Files to create

- `app/routers/health.py`

### Notes

- The full endpoint path is `/v1/health` (prefix on the `APIRouter`, not a nested prefix at app-mount time).
- Health intentionally uses the envelope per ADR-002 — do not return a naked `{"status": "ok"}`.
- Engine version is sourced from the `HealthData` default (which reads from `app/core/config.ENGINE_VERSION`); do not duplicate the literal here.
