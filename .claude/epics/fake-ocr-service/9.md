# Task: App factory

## Description

Wire the pieces together in `app/main.py`: configure logging on startup, register the request-ID middleware, mount both routers, and expose a module-level `app` for `uvicorn`. This is the join point of the three DAG branches (logging+middleware, health router, OCR router). Implements [[2026-04-26-layered-project-structure]]'s factory placement.

## Acceptance Criteria

- [ ] `app/main.py` exposes `create_app() -> FastAPI` that:
  - Calls `configure_logging()` once during application lifespan startup (via FastAPI's lifespan context manager).
  - Constructs `FastAPI(title="fake-ocr-service", version=ENGINE_VERSION)`.
  - Registers `RequestIDMiddleware` via `app.add_middleware(...)`.
  - Includes both routers: `app.include_router(health.router)` and `app.include_router(ocr.router)`.
  - Returns the configured app.
- [ ] Module-level `app = create_app()` so `uv run uvicorn app.main:app` resolves correctly.
- [ ] No business logic, no constants, no schemas defined in `main.py` — only wiring (per [[2026-04-26-layered-project-structure]]).
- [ ] **Gate (mirrors Phase I):** `uv run uvicorn app.main:app` starts without error and `curl http://localhost:8000/v1/health` returns:
  - status `200`
  - header `X-Request-ID` set (UUIDv4 hex when no ID is sent)
  - body matches the documented health envelope

## Technical Details

### Files to create

- `app/main.py`

### Notes

- Use FastAPI lifespan (`@asynccontextmanager`) to call `configure_logging()` at startup. Do not call it at module import time — that creates side effects on `import app.main` from tests.
- Middleware order: register `RequestIDMiddleware` first (outermost). It must observe every response, including those from later-registered framework middleware.
- The `version=ENGINE_VERSION` ties the FastAPI app version to the engine version on purpose. Bumping `ENGINE_VERSION` (a code change per ADR-001) automatically bumps the OpenAPI `info.version` field.
