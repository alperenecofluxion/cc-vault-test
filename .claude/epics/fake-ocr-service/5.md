# Task: Pydantic envelope schemas

## Description

Define the response envelope contract in `app/schemas.py` using Pydantic v2 generic models. All endpoints, on every code path, return either `SuccessResponse[T]` or `ErrorResponse`. This task is the canonical implementation of [[2026-04-26-api-response-envelope-shape]] — the envelope is enforced server-side by these models so that drift is impossible at the route level.

## Acceptance Criteria

- [ ] `app/schemas.py` defines (Pydantic v2):
  - `OCRData(BaseModel)`: `text: str`, `extension: str`, `filename: str`, `engine: str = ENGINE_VERSION`.
  - `HealthData(BaseModel)`: `status: str`, `engine: str = ENGINE_VERSION`.
  - `ErrorDetail(BaseModel)`: `code: str`, `message: str`.
  - `SuccessResponse(BaseModel, Generic[T])`: `success: Literal[True] = True`, `data: T`, `error: None = None`.
  - `ErrorResponse(BaseModel)`: `success: Literal[False] = False`, `data: None = None`, `error: ErrorDetail`.
- [ ] All `engine` fields default to `app.core.config.ENGINE_VERSION` (imported, not duplicated).
- [ ] Models use `model_config = ConfigDict(...)` with whatever Pydantic v2 settings the project standardises on (e.g., `extra="forbid"` to catch typos).
- [ ] **Gate (mirrors Phase D):** the following round-trips and produces the exact envelope shape:
  ```python
  SuccessResponse[OCRData](data=OCRData(text="x", extension="pdf", filename="a.pdf")).model_dump()
  # → {"success": True, "data": {"text": "x", "extension": "pdf", "filename": "a.pdf", "engine": "fake-v0"}, "error": None}
  ```
- [ ] An equivalent `ErrorResponse(error=ErrorDetail(code="x", message="y")).model_dump()` produces `{"success": False, "data": None, "error": {"code": "x", "message": "y"}}`.

### Notes

- Pydantic v2 generics: `SuccessResponse[T]` requires `Generic[T]` and `TypeVar`. Use `from typing import Generic, Literal, TypeVar`.
- `Literal[True] / Literal[False]` on the `success` flag makes the discriminator explicit and lets clients narrow with type checkers.
- Both `data: None = None` on error and `error: None = None` on success ensure the JSON shape is identical across paths — clients always see all three keys.
- Do NOT add fields beyond what is listed. The PRD §FR-2 success payload has exactly four fields; do not invent `confidence`, `bounding_boxes`, etc.
